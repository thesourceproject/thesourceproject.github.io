Handling user input in Java

21 Mar '21 | 15 min read

#basics #java #beginner

In Java, there are two main classes for dealing with user input, namely Scanner and BufferedReader. But which one is better? In this tutorial, we will learn how to use both. If you're planning to go into competitive programming, this tutorial is a must for you.

Getting Started

This tutorial assumes that you have some basic understanding of Java, particularly data-types. Before we dive into the code, first we need to understand how input works. When you type in characters in the console and press Enter, your input(including the newline character at the end) goes to the program as a stream. The Scanner and BufferedReader classes process this stream of characters in a slightly different manner.

Scanner

Let's start with an example of the Scanner class. It is present in the java.util package. To use it, we need to create an instance of the Scanner class tied to System.in(input stream which is typically connected to the keyboard input of console programs). 

import java.util.Scanner; // Import the Scanner class.
class JavaInput {
    public static void main(String args[]) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter your name: ");
        String name = sc.next(); // John
        System.out.println("Hello " + name); // Hello John
    }
}

The next() method of the Scanner class reads in the stream of characters until it encounters any whitespace(including newline). It then returns those characters as a string(without the terminating newline).
We cannot use this method if we want to input a string with any spaces in it. Try to run it again with multiple words. It will only capture the first word and ignore the rest. For this, we have another method named nextLine().
Note - If your IDE(text-editor) gives you a warning like Resource leak: 'sc' is never closed, you can safely ignore it. Unlike file streams, resources tied to System.in are not required to be closed. 

import java.util.Scanner; // Import the Scanner class.
class JavaInput {
    public static void main(String args[]) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter your name: ");
        String name = sc.nextLine(); // John Smith
        System.out.println("Hello " + name); // Hello John Smith
    }
}

What if we need to input numbers and not strings. The Scanner class has a number of methods for capturing primitive data-types. These work similar to the next() method by returning the next token(a group of non-whitespace characters). The following are the most commonly used ones.

  • nextInt()
  • nextLong()
  • nextFloat()
  • nextDouble()
  • nextBoolean() - Only accepts true or false.

Note - These functions will throw an InputMismatchException if the input is of an unexpected type. 

import java.util.Scanner; // Import the Scanner class.
class JavaInput {
    public static void main(String args[]) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter random data: "); // Input can be given in single or multiple lines.
        int i = sc.nextInt(); // 29
        long l = sc.nextLong(); // 4921
        float f = sc.nextFloat(); // 0.5
        double d = sc.nextDouble(); // 2.57843
        boolean b = sc.nextBoolean(); // false
        System.out.println(i + " " + l + " " + f + " " + d + " " + b); // 29 4921 0.5 2.57843 false
    }
}

Scanner does not have a separate method to read in a character. Instead, we can use the next() method to input a single character string and use charAt(0) to get the first and only character. 

import java.util.Scanner; // Import the Scanner class.
class JavaInput {
    public static void main(String args[]) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter a single character: ");
        char c = sc.next().charAt(0); // A
        System.out.println(c); // A
    }
}

BufferedReader

The Scanner class provides an easy-to-remember way of taking user input but it has one major disadvantage. It performs poorly when there is a very large amount of data. This becomes a determining factor in competitive programming. BufferedReader, due to its larger buffer size, performs much better.
It is also synchronized, meaning that it is thread-safe, when working in a multi-threading environment.

Let's look at a basic example of BufferedReader. 

import java.io.*; // Import the InputStreamReader, BufferedReader, and IOException classes.
class JavaInput {
    public static void main(String args[]) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        System.out.print("Enter you name: ");
        String name = br.readLine(); // John Smith
        System.out.println(name); // John Smith
    }
}

The syntax for BufferedReader is slightly more difficult to remember. One notable difference is that the readLine() and read() methods of BufferedReader throw IOException. Therefore, it must either be caught(in a try-catch) or thrown by main().
The readLine() method works similar to the nextLine() method of Scanner and does not split the input into tokens.
Unlike Scanner, BufferedReader does not do any parsing of the input. This means we have to manually convert the returned string to our desired data-type using their wrapper class functions.
Note - Exception handling in Java will be covered in detail in a later tutorial.

  • Integer.parseInt()
  • Long.parseLong()
  • Float.parseFloat()
  • Double.parseDouble()
  • Boolean.parseBoolean() - will only return true only if the string is exactly true, otherwise false.

Note - These functions will throw a NumberFormatException if the input is of an unexpected type. 

import java.io.*; // Import the InputStreamReader, BufferedReader, and IOException classes.
class JavaInput {
    public static void main(String args[]) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        System.out.println("Enter random data:"); // Input has to be given using multiple lines.
        int i = Integer.parseInt(br.readLine()); // 29
        long l = Long.parseLong(br.readLine()); // 4921
        float f = Float.parseFloat(br.readLine()); // 0.5
        double d = Double.parseDouble(br.readLine()); // 2.57843
        boolean b = Boolean.parseBoolean(br.readLine()); // false
        System.out.println(i + " " + l + " " + f + " " + d + " " + b); // 29 4921 0.5 2.57843 false
    }
}

Unlike Scanner, BufferedReader has a method for reading in a single character(as an int, not char) named read(). But use this method carefully as it does not ignore the terminating newline character. To understand this, run the following. 

import java.io.*; // Import the InputStreamReader, BufferedReader, and IOException classes.
class JavaInput {
    public static void main(String args[]) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        System.out.println("Enter first character:");
        char c1 = (char) (br.read()); // Typecast int to char.
        System.out.println("Enter second character:");
        char c2 = (char) (br.read());
        System.out.println(c1 + " " + c2);
    }
}

When you type in your first character and press Enter, the program doesn't prompt you for the second time. This is because the terminating newline in your first input gets counted as the second character.
To avoid this problem, you can use the same idea used for Scanner i.e. taking input using readLine() and obtain its first character using charAt(0).

In competitive programming, the input is usually given in the form of space-separated integers or strings. The Scanner class has methods to directly parse these tokens. But how can this be done with BufferedReader? The StringTokenizer class will help us here. 

import java.io.*; // Import the InputStreamReader, BufferedReader, and IOException classes.
import java.util.StringTokenizer; // Import the StringTokenizer class.
class JavaInput {
    public static void main(String args[]) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        System.out.println("Enter numbers separated by comma:");
        StringTokenizer st = new StringTokenizer(br.readLine(), ", "); // 10, 20, 30, 40 (Tokenize the string using ", " as the delimiter)
        int sum = 0;
        while (st.hasMoreTokens()) {
            sum += Integer.parseInt(st.nextToken());
        }
        System.out.println(sum); // 100
    }
}

The StringTokenizer() constructor takes in two arguments. The original string and the delimiter(optional, whitespace by default). The hasMoreTokens() method checks whether there are any tokens remaining and nextToken() returns it as a string. It is then up to us how to parse it.

That's all you need to know about user input in Java. Now try to use the Scanner and BufferedReader classes in your programs and see which one suits your use-case better.
