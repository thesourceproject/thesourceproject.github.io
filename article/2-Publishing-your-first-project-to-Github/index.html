Publishing your first project to Github

4 Jan '21 | 10 min read

#miscellaneous #git #github #beginner

In our previous tutorial, we learned how to create a local Git repository and make commits to it. Now it's time to show off your first project to the world.

GitHub is a code hosting platform for version control and collaboration. It allows you to showcase your work and lets you and others work together on projects from anywhere.

Getting Started

Note - You need to have Visual Studio Code and local Git repository set up before you can continue. You may refer to this tutorial for instructions on how to install VS Code and create a local Git repository.
Sign up on Github. After completing the sign-up process, open your project folder in VS Code.
Click the Publish to Github button in the status bar on the bottom.

A prompt will appear asking The extension 'Github' wants to sign in using Github
Click Allow.

Your browser will be opened automatically and you will receive another prompt asking you to authorize Visual Studio Code to access Github.
Click Continue.

You will then see a prompt asking you to allow an extension to open this URI.
Click Open.

Once the authentication is complete you will be asked by VS Code to set a repository name for Github. By default, it is set to the current folder name.
You are also given two options, whether to publish to a private repository or a public repository. If you want your repository to be private i.e. only be seen by you, you can select Publish to Github private repository.
But for now, select Publish to Github public repository.

You will be prompted to sign in to Github.
Click Sign in with your browser to continue.

Your browser will show you that the authentication has succeeded. Close the tab and return to VS Code.

You will see a notification below saying that the repository has been published successfully. You may click the Open in Github button to open the repository in your browser
It may also show you another notification asking you to allow VS Code to periodically run git fetch. You can ignore this for now.

And there you go! You have published your first project to Github. Now you are ready to showcase your work to the world!
