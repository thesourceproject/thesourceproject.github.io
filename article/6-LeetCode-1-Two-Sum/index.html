LeetCode #1 Two Sum

20 Feb '21 | 10 min read

#competitive #data #structures #algorithms #java #python #beginner

Check out this problem on LeetCode.

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.
You may assume that each input would have exactly one solution, and you may not use the same element twice.
You can return the answer in any order.

Getting Started

We need to find a pair of numbers in the given array that adds up to target. The most straightforward way to find it is to try every possible pair. 

// Java
public int[] twoSum(int[] nums, int target) {
    for(int i = 0; i < nums.length; i++) {
        for(int j = 0; j < nums.length; j++) {
            if(i != j) { // You may not use the same element twice.
                if(nums[i] + nums[j] == target) {
                    return new int[] {i, j}; // Return the indices in an array.
                }
            }
        }
    }
    return new int[] {-1, -1}; // Default return value if there is no solution.
}
# Python3
def twoSum(self, nums: List[int], target: int) -> List[int]:
    for i in range(len(nums)):
        for j in range(len(nums)):
            if(i != j):
                if(nums[i] + nums[j] == target):
                    return [i, j]

Notice that we added a default return value in the Java code at the end. This is because the return type of the function is of int[] type, therefore the function must return an array of integers for all conditions. This is not required in the Python version as the question states that we will get a unique solution for every test case.

This solution has a time complexity of O(n2) because we run 2 nested loops which each run for the complete length of the array in the worst case. If we submit this, it will get accepted. But we can improve upon this solution slightly.

If you see our code, it runs 2 loops from the beginning and we are checking every pair 2 times. For example, initially we check the pair (i, j) = (0, 1) and when the inner loop completes once, we check the pair (i, j) = (1, 0). The sum is going to be the same in either order so we can save some time by skipping these pairs.
Let's see how we can achieve this.

Suppose there's an array [0, 1, 2, 3, 4]. In the first outer loop we will check the pairs (0, 1), (0, 2), (0, 3), (0, 4). For the next loop, we can start from (1, 2) instead of (1, 0) and continue (1, 3), (1, 4). And then (2, 3), (2, 4) and finally (3, 4).
We observe that we can start the inner loop from i + 1 to avoid the visited pairs. 

// Java
public int[] twoSum(int[] nums, int target) {
    for(int i = 0; i < nums.length; i++) {
        for(int j = i + 1; j < nums.length; j++) {
            if(nums[i] + nums[j] == target) {
                return new int[] {i, j};
            }
        }
    }
    return new int[] {-1, -1};
}
# Python3
def twoSum(self, nums: List[int], target: int) -> List[int]:
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if(nums[i] + nums[j] == target):
                return [i, j]

This solution like the previous one has a space complexity of O(1) as we only use a constant amount of memory to store our variables. Notice now we don't have to check for i != j as i and j will never be equal.

We improved our solution slightly but it's time complexity is still O(n2). This is because in the worst case our solution checks (n - 1) + (n - 2) + (n - 3) + ... + 2 + 1 = (n)(n - 1) รท 2 pairs, thereby giving it an upper bound of O(n2).

Can we bring our solution down to O(n)?
Yes! We can. Here comes HashMap/Dictionary to the rescue.

Java Hashmaps & Python Dictionaries can store key-value pairs that allow lookup in O(1) time. But how will it help us? For every number, if we could check whether its difference from target exists in the array, we will get the answer.
Note - You can learn more about HashMaps and Dictionaries in our tutorials. 

// Java
// import java.util.*; // For the HashMap class, it is automatically imported on LeetCode.
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> indices = new HashMap<>();
    for(int i = 0; i < nums.length; i++) {
        if(indices.containsKey(target - nums[i])) {
            return new int[] {indices.get(target - nums[i]), i};
        }
        indices.put(nums[i], i);
    }
    return new int[] {-1, -1};
}
# Python3
def twoSum(self, nums: List[int], target: int) -> List[int]:
    indices = {};
    for i in range(len(nums)):
        if((target - nums[i]) in indices):
            return [indices[target - nums[i]], i]
        indices[nums[i]] = i

Here we create a HashMap/Dictionary to store the indices of the array elements. Then we traverse over the array while checking if we have encountered target - nums[i] previously. If not, we store the index with the array element as the key in our HashMap/Dictionary.

So now we have a solution with a time complexity of O(n) making it much faster than the previous ones, however it also has a space complexity of O(n) because we are using additional memory for the HashMap/Dictionary proportional to the size of the input array in the worst case.
